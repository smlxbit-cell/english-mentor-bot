# PRODUCT DESIGN — English Mentor Bot

> Методическая и продуктовая часть: КАК устроено обучение, а не только код.
> Всё рассчитано на русскоязычного ученика и на экономию AI-токенов.

---

## 1. Путь пользователя (User Journey)

```
/start
  → Тёплое приветствие + короткий онбординг (2 вопроса: цель и интересы)
  → Бесплатная адаптивная ДИАГНОСТИКА (5–8 адаптивных заданий, ~3 мин)
  → Результат: уровень (A1–B2) + сильные/слабые стороны + план
  → ПРОБНЫЙ УРОК 1  «Кофейня в Лондоне»   (интерактив, медиа, AI, XP)
  → ПРОБНЫЙ УРОК 2  «Знакомство в самолёте» (интерактив, диалог с AI, голос)
  → PAYWALL: подписка 390 ₽ / 30 дней
  → (после оплаты) Ежедневные адаптивные сессии + словарь (SRS) + сериал
```

Принцип: **каждый шаг должен давать маленькую победу** (правильный ответ → XP →
анимация/эмодзи → «ты молодец»). Мотивация важнее объёма.

## 2. Адаптивная диагностика (профессиональная, но лёгкая)

Цель — за 5–8 заданий определить уровень **A1 / A2 / B1 / B2** и собрать профиль
слабых зон (грамматика, лексика, восприятие на слух, говорение).

### Механика (адаптивный алгоритм, без AI — экономно)
- Банк заданий `DiagnosticItem`, у каждого: `level` (a1..b2), `skill`, тип, ответ.
- Старт с **A2** (середина). Дальше — «лесенка»:
  - ответил верно → следующий вопрос на уровень выше;
  - ошибся → уровень ниже;
  - алгоритм сходится к уровню, где ~50–70% верных.
- Проверка диагностики — **детерминированная** (варианты/точное совпадение/ключевые
  слова). AI НЕ вызываем — это дёшево и быстро.
- Итог: `level` = точка сходимости; по `skill`-статистике формируем «слабые зоны».

### Типы заданий в диагностике
1. **Multiple choice** (грамматика): `She ___ coffee every morning. (drink / drinks / drinking)`
2. **Fill gap** (короткий ввод): `If I ___ time, I would travel.`
3. **Listening** (аудио → выбор): проигрываем короткое аудио, спрашиваем смысл.
4. **Перевод RU→EN** (проверка по ключевым словам, не строгая).
5. **Speaking (голос)**: «Скажи вслух: *I would like a coffee, please.*» → STT
   (Yandex SpeechKit) → сверяем ключевые слова/похожесть. Опционально, влияет на
   оценку `pronunciation/speaking`, но не блокирует.

### Экран результата (пример)
```
Готово! Твой уровень: A2 (Pre-Intermediate) 🎯

Сильные стороны: базовая лексика, Present Simple.
Над чем поработаем: прошедшее время, восприятие на слух.

Я собрал для тебя персональный план. Начнём с 2 бесплатных уроков →
```

## 3. Модель интерактивного урока

Урок = последовательность **шагов** (`LessonStep`). Бот ведёт по одному шагу,
запоминая позицию. Типы шагов (охватывают всю «фишечность»):

| Тип шага | Что делает |
|----------|-----------|
| `hook` | Цепляющее вступление, интрига, зачем это ученику |
| `story` | Кусочек истории/сцена с персонажем (текст + картинка) |
| `image` / `gif` | Медиа: фото/GIF с подписью |
| `audio` | Аудио: фраза/диалог для прослушивания |
| `video` | Видео-фрагмент |
| `vocabulary` | Карточки слов: слово — перевод — пример — (аудио) |
| `grammar_note` | Мини-правило простыми словами (по-русски) |
| `dialogue` | Диалог: реплики бота + выбор/ввод ученика |
| `exercise` | Задание (см. §4) |
| `speaking` | Голосовое задание (STT), сравнение с образцом |
| `ai_dialogue` | Свободный диалог с AI-персонажем (роль, экономный бюджет) |
| `reflection` | «Что нового узнал», лёгкий итог |
| `reward` | Начисление XP/бонуса, поздравление |
| `cliffhanger` | Интрига → мотивация вернуться к следующему уроку |

Каждый шаг может нести `MediaAsset` и/или `Character`. `file_id` из Telegram
кэшируется в `MediaAsset.telegram_file_id`, чтобы не грузить файл повторно.

## 4. Типы упражнений и проверка (ЭКОНОМИЯ AI)

Ключевая идея экономии: **AI вызываем только там, где иначе никак.**

| Тип | Как проверяем | AI? |
|-----|---------------|-----|
| `multiple_choice` | сравнение выбранной опции | ❌ нет |
| `true_false` | сравнение | ❌ нет |
| `fill_gap` | нормализация + список допустимых вариантов | ❌ нет |
| `word_order` | сравнение порядка | ❌ нет |
| `matching` | сравнение пар | ❌ нет |
| `translation` | нормализация + **ключевые слова** (must-have леммы) | ❌ нет* |
| `short_answer` | ключевые слова | ❌ нет* |
| `writing` (свободный текст) | **AI**: оценка + исправления + 1 совет | ✅ да |
| `dialogue_simulation` | **AI**: играет роль, коротко реагирует | ✅ да |
| `speaking` | STT → ключевые слова/похожесть; AI по желанию | ⚠️ STT, AI опц. |

\* Для перевода/коротких ответов сначала пытаемся детерминированно (ключевые
слова). Если не сошлось И задание помечено `ai_fallback=true` — один короткий
AI-запрос как «второе мнение».

### Стратегия экономии токенов (реализована в `ai_app`)
1. **Deterministic-first**: MC/gap/order/matching — 0 токенов.
2. **Дешёвая модель** `gpt-4o-mini`, короткие системные промпты, `max_tokens` мал.
3. **Жёсткий формат ответа** (JSON: `is_correct`, `score`, `feedback_ru`,
   `correction`, `tip_ru`) — без «воды».
4. **Кэш** одинаковых (упражнение + нормализованный ответ) → повторно бесплатно.
5. **Дневной бюджет** AI-вызовов на пользователя (лимит в settings). Превышен →
   деградация в детерминированную проверку + вежливое сообщение.
6. **Батч подсказок**: подсказки к заданию пишет автор заранее (`hints`), AI —
   только если автор не задал.
7. **Диалог с AI** — ограничен по числу реплик и длине контекста (окно 4–6 реплик).

Итого: платный трафик AI ≈ только writing/dialogue, что легко умещается в 390 ₽.

## 5. Геймификация (удержание мотивации)

- **XP** за каждый шаг/правильный ответ; уровни ученика (`UserStats.level`).
- **Очки/бонусы**: серия правильных ответов подряд = множитель («комбо»).
- **Стрик** (дни подряд): напоминания, «не потеряй стрик 🔥».
- **Достижения** (`Achievement`): «Первый шаг», «3 дня подряд», «100 слов»,
  «Первый диалог с AI», «Идеальный урок (без ошибок)».
- **Прогресс-бар урока** и экран итогов: «5/5 верно, +60 XP, стрик 4🔥».
- **Ежедневная цель** (`daily_minutes`) — маленькая, достижимая.
- Дальше: недельные челленджи, лидерборд среди друзей, «магазин» бонусов
  (разблокировка бонусных сцен сериала за очки).

## 6. Словарь + интервальное повторение (SRS)

- Слова живут в `learning.Word` (english, translation, example, difficulty).
- Прогресс — `progress_app.UserWordProgress` (status, strength, next_review_at).
- **SRS**: простой алгоритм (Leitner/SM-2-lite): верный ответ → интервал растёт,
  ошибка → сброс. `next_review_at` планирует повтор.
- Тренировки слов в боте:
  - **перевод** (EN→RU / RU→EN),
  - **слушай слово** (аудио/TTS) → выбери значение,
  - **слово в контексте** (пример-предложение с пропуском),
  - **разные переводы/значения** (полисемия — из `Phrase`/примеров).
- Новые слова автоматически попадают в словарь из пройденных уроков.

## 7. Персонаж и «мини-сериал»

- `content_app.Character` + `StoryArc` + `StoryEpisode` (уже есть).
- Сквозной персонаж-гид (напр. **Emma**) сопровождает ученика: реплики, аватар,
  озвучка. Позже — генерация изображений персонажа под сцены.
- Сериал: каждая новая серия = урок с продолжением истории и `cliffhanger`.
- AI-персонаж в `ai_dialogue`: системный промпт задаёт роль/характер
  (`Character.personality`, `speaking_style`), отвечает коротко и по уровню ученика.

---

## 8. Авторские права (осторожно!)

- **Не использовать** защищённые тексты/аудио/видео (фильмы, песни, учебники) без прав.
- Тексты историй/диалогов — **оригинальные**, пишем сами / генерируем.
- Медиа — только: собственное, сгенерированное AI, или лицензии CC0/CC-BY (с
  указанием автора). В `MediaAsset` хранить `license_name`/`author`/`source_url`.
- «Кусочки литературы» — брать только **public domain** (напр. классика, срок
  истёк) и помечать источник. Современное — нельзя.

## 9. Флагманский пробный урок 1 — «Coffee in London» (A1–A2)

Показывает всю суть продукта за ~7–8 минут.

1. `hook`: «Ты в Лондоне. Хочешь кофе, но стесняешься заговорить. Через 5 минут ты
   закажешь его как местный ☕️» + фото кофейни.
2. `story`: Emma заходит в кофейню. Короткая сцена (картинка + текст).
3. `vocabulary`: карточки — *coffee, tea, cup, please, would like* (перевод + пример
   + аудио).
4. `grammar_note` (по-русски): вежливое «I would like…» вместо «I want…».
5. `audio`: прослушай диалог у стойки.
6. `exercise` (MC): «Как вежливо попросить кофе?» → *I would like a coffee, please.*
7. `exercise` (fill gap): `I ___ like a tea, please.` → *would*.
8. `speaking` (голос): «Скажи: *A cup of coffee, please.*» → STT → фидбэк.
9. `dialogue` (мини): бариста спрашивает — ученик выбирает/пишет ответ.
10. `reward`: +60 XP, «Готово! Ты сделал первый заказ 🎉».
11. `cliffhanger`: «А что, если тебя спросят про размер и молоко? Это в уроке 2…»

## 10. Флагманский пробный урок 2 — «Meeting on a Plane» (A2)

Добавляет свободный диалог с AI и говорение.

1. `hook`: «Соседка в самолёте заговорила с тобой по-английски. Не растеряйся!»
2. `story`: Emma садится рядом, улыбается (картинка).
3. `vocabulary`: *seat, flight, trip, nice to meet you, where are you from*.
4. `grammar_note`: вопросы в Present Simple (*Where do you…?*).
5. `exercise` (matching): вопрос ↔ ответ.
6. `exercise` (word order): собери *«Where are you from?»*.
7. `ai_dialogue`: свободный small talk с Emma (AI-роль, 4–5 реплик, экономно).
   AI мягко исправляет и хвалит.
8. `speaking`: «Представься: скажи, как тебя зовут и откуда ты».
9. `reflection`: 3 фразы, которые теперь умеешь.
10. `reward`: +80 XP, достижение «Первый диалог с AI 🗣️».
11. Переход к **paywall**: «Дальше — 30 дней ежедневных приключений с Emma за 390 ₽».
